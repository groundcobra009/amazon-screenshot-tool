## 1. このツールでできること

Amazon Kindleアプリで開いた本を、自動でページ送りしながらスクリーンショットを撮影し、1つのPDFファイルにまとめるツールです。

- 自動でページをめくりながらスクリーンショットを撮影
- 最後のページまで行ったら自動で停止
- 撮影した画像を1つのPDFにまとめて出力
- 途中で停止することも可能

## 2. 必要なもの

| 項目 | 詳細 |
|------|------|
| Mac | macOS Ventura（13）以降 |
| Amazon Kindle | App Storeからインストール |
| Homebrew | macOSのパッケージマネージャー |
| Python | 3.10以降（Homebrewでインストール可能） |
| ターミナル | Terminal.app / iTerm / VS Code等 |

### Homebrewが入っていない場合

ターミナルを開いて以下を実行:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### Pythonが入っていない場合

```bash
brew install python
```

## 4. macOS権限の設定

このツールはKindleアプリの操作とスクリーンショット撮影のため、macOSの権限設定が必要です。

### 4-1. アクセシビリティ権限

**何のため**: Kindleアプリへのキー送信（ページめくり）に必要

1. 「システム設定」を開く
2. 「プライバシーとセキュリティ」をクリック
3. 「アクセシビリティ」をクリック
4. ツールを起動するターミナルアプリ（Terminal / iTerm等）をONにする

<!-- [スクリーンショット] システム設定 → アクセシビリティの画面（ターミナルがONになっている状態） -->

### 4-2. 画面収録権限

**何のため**: Kindleウィンドウのスクリーンショット撮影に必要

1. 「システム設定」を開く
2. 「プライバシーとセキュリティ」をクリック
3. 「画面収録」をクリック
4. ツールを起動するターミナルアプリをONにする

<!-- [スクリーンショット] システム設定 → 画面収録の画面（ターミナルがONになっている状態） -->

> **重要**: 権限を変更した後はターミナルアプリを再起動してください。

## 5. ツールの起動方法

ターミナルで以下の2行を実行:

```bash
cd amazon-screenshot-tool
source .venv/bin/activate && python main.py
```

GUIウィンドウが画面右上にフローティング表示されます。

## 6. 画面の見方と設定項目

<!-- [スクリーンショット] GUI画面に番号付きの注釈を入れたもの（以下の①〜⑥に対応） -->

| 番号 | 項目 | 説明 |
|------|------|------|
| ① | Mac画面サイズ | お使いのMacのインチ数を選択（13/14/15/16インチ） |
| ② | めくり方向 | 「右めくり（←で次ページ）」が一般的。縦書き本は「左めくり（→で次ページ）」 |
| ③ | 本の名前 | 保存フォルダ名・PDFファイル名になります。空欄だと日時で自動命名 |
| ④ | ページ送り待機(秒) | ページめくり後、次のスクショまでの待ち時間。画像が多い本は2〜3秒推奨 |
| ⑤ | テスト撮影 | チェックすると5枚だけ撮影して停止。初回の動作確認用 |
| ⑥ | 撮影開始 / 停止 | 撮影の開始と途中停止 |

### めくり方向の選び方

| 本の種類 | めくり方向 |
|----------|-----------|
| 横書き（ビジネス書、技術書等） | 左めくり（→ で次ページ） |
| 縦書き（小説、漫画等） | 右めくり（← で次ページ） |

> **迷ったら**: Kindleで手動で1ページめくってみて、どのキーで進むか確認してください。

## 7. 撮影の手順（ステップバイステップ）

### STEP 1: Kindleで本を開く

1. Amazon Kindleアプリを起動
2. 撮影したい本を開く
3. **最初のページ（表紙）に移動**
4. Kindleを**フルスクリーン**にする（メニューバー「表示」→「フルスクリーンにする」またはCmd+Ctrl+F）

<!-- [スクリーンショット] Kindleでフルスクリーン表示した状態 -->

### STEP 2: ツールを起動・設定

1. ターミナルでツールを起動（`source .venv/bin/activate && python main.py`）
2. GUIで設定を入力:
   - めくり方向を選択
   - 本の名前を入力
   - ページ送り待機時間を設定（通常1.5秒でOK）

<!-- [スクリーンショット] 設定入力済みのGUI画面 -->

### STEP 3: 撮影開始

1. 「撮影開始」ボタンをクリック
2. Kindleが自動でアクティブになる
3. 3秒のカウントダウン後、自動撮影が始まる

<!-- [スクリーンショット] 撮影中のGUI（ログにページ番号が増えている状態） -->

### STEP 4: 自動完了を待つ

- ログ欄でページ数が増えていくのを確認
- 最終ページに到達すると自動停止
- PDFが自動生成される

<!-- [スクリーンショット] 完了ダイアログ「撮影完了: XXページ」 -->

### STEP 5: PDFを確認

`output/` フォルダにPDFが保存されています。

```
amazon-screenshot-tool/
└── output/
    └── 本の名前.pdf    ← ここ
```

## 8. テスト撮影で動作確認する

初めて使うときは、まずテスト撮影で動作確認してください。

1. 「テスト撮影（5枚のみ）」にチェックを入れる
2. 「撮影開始」を押す
3. 5枚撮影して自動停止する
4. `screenshots/本の名前/` フォルダで画像を確認

<!-- [スクリーンショット] テスト撮影にチェックが入ったGUI画面 -->

### 確認ポイント

- [ ] スクリーンショットにKindleの本文だけが映っている（メニューバーやGUIが映っていない）
- [ ] ページが正しく送られている（5枚とも違うページ）
- [ ] 画像がきれいに撮れている

問題があれば「10. トラブルシューティング」を参照してください。

## 10. トラブルシューティング

### 「Kindleアプリが起動していません」と表示される

- Amazon Kindleアプリを起動してください
- 本を開いた状態にしてください

### 「Kindleウィンドウが見つかりません」と表示される

- Kindleで本が開かれているか確認
- Kindleウィンドウが最小化されていないか確認

### ページが送られない（全部同じ画像になる）

- **アクセシビリティ権限**がターミナルに付与されているか確認
- Kindleを**フルスクリーンにしてから**撮影開始してください
- めくり方向が正しいか確認（横書き→左めくり、縦書き→右めくり）

<!-- [スクリーンショット] アクセシビリティ権限の設定画面 -->

### スクリーンショットが真っ黒

- **画面収録権限**がターミナルに付与されているか確認
- 権限変更後はターミナルを再起動してください

### tkinterエラー（`No module named '_tkinter'`）

お使いのPythonバージョンを確認して:

```bash
python3 --version
```

対応するtkinterをインストール:

```bash
brew install python-tk@3.XX
```

（`3.XX`はお使いのPythonバージョンに合わせてください）

### 画像が多い本でページが飛ぶ

- 「ページ送り待機(秒)」を **2〜3秒** に増やしてください
- 画像の多い本はページの描画に時間がかかります
