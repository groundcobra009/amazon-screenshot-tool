# Kindle Screenshot Tool - 要件定義書

## 概要

macOS上で動作するローカルアプリケーション。Amazon Kindleアプリの画面を自動でページ送りしながらスクリーンショットを撮影し、最終的にPDFとしてまとめる。個人利用を前提とする。

---

## 利用フロー

1. ユーザーがKindleアプリを起動し、撮影したい本を開く
2. 本ツールを起動する
3. ツールが初期設定を対話的に確認する（ページめくり方向など）
4. ツールがKindleウィンドウをアクティブ化 → フルスクリーン化
5. 自動でスクリーンショット撮影 → ページ送り → 撮影 を繰り返す
6. 画面に変化がなくなったら自動停止
7. 撮影した画像をPDFにまとめて出力

---

## 機能要件

### F1: 初期設定（対話入力）

| 項目 | 内容 |
|------|------|
| Macの画面サイズ | 内蔵ディスプレイのインチ数を確認（13/14/15/16インチ等） |
| キャプチャ対象 | 常にMacの内蔵ディスプレイを使用（外部モニターではなくMac画面にKindleを表示するよう案内） |
| ページめくり方向 | 右めくり（←キーで次ページ）/ 左めくり（→キー で次ページ） |
| 保存ディレクトリ | プロジェクト内にbook名でディレクトリ作成 |
| book名 | 最初のスクリーンショット（表紙）から自動命名、またはユーザー入力 |

### F2: ディスプレイ確認・案内

- ユーザーにMacの画面サイズ（インチ）を確認
- 「KindleアプリをMacの内蔵ディスプレイに移動してください」と案内表示
- キャプチャは常にMac内蔵ディスプレイ（メインディスプレイ）を対象とする
- 外部モニター環境でも動作を安定させるため、対象を内蔵に固定

### F3: Kindleウィンドウ制御

- Kindleアプリウィンドウを検出しアクティブ化
- Mac内蔵ディスプレイ上でフルスクリーンに切り替え
- macOS Accessibility API（またはAppleScript）を使用

### F3: スクリーンショット撮影

- アクティブなKindleウィンドウのキャプチャ
- 連番ファイル名で保存（例: `001.png`, `002.png`, ...）
- 保存先: `./screenshots/{book_name}/`

### F4: 自動ページ送り

- 指定方向のキー入力をKindleに送信
- ページ送り後、描画完了を待ってからキャプチャ
- 待機時間は設定可能（デフォルト: 1〜2秒）

### F5: 自動停止判定

- 前回のスクリーンショットと今回のスクリーンショットを画像比較
- 変化がない（= 最終ページ到達）と判定したら停止
- 比較方法: ピクセル差分 or 画像ハッシュ比較
- 連続N回（デフォルト2回）変化なしで確定停止

### F6: PDF生成

- 撮影した全画像を1つのPDFにまとめる
- ファイル名: `{book_name}.pdf`
- ページ順序は撮影順

## 非機能要件

| 項目 | 内容 |
|------|------|
| 対応OS | macOS（Ventura以降） |
| 言語 | Python 3.10+ |
| 権限 | アクセシビリティ権限（画面録画権限も必要な場合あり） |
| 依存管理 | pip / requirements.txt |
| 実行方式 | CLIツール（ターミナルから実行） |

---

## 技術スタック

| 用途 | ライブラリ/技術 |
|------|----------------|
| ウィンドウ制御 | `pyobjc` (AppKit/Quartz) または `osascript`（AppleScript） |
| スクリーンショット | `Quartz` (CGWindowListCreateImage) または `screencapture` コマンド |
| キー入力送信 | `pyobjc` (CGEvent) または AppleScript |
| 画像比較 | `Pillow` + NumPy（ピクセル差分）または `imagehash` |
| PDF生成 | `img2pdf` または `Pillow`（PDF保存） |
| CLI対話 | 標準入力 or `click` / `inquirer` |

---

## macOS権限について

本ツールの動作に以下のシステム権限が必要:

1. **アクセシビリティ** - キー入力送信、ウィンドウ制御に必要
   - システム設定 → プライバシーとセキュリティ → アクセシビリティ → ターミナル(またはiTerm)を許可
2. **画面収録** - スクリーンショット撮影に必要
   - システム設定 → プライバシーとセキュリティ → 画面収録 → ターミナルを許可

## ディレクトリ構成（想定）

```
amazon-screenshot-tool/
├── main.py              # エントリーポイント
├── kindle_controller.py # Kindleウィンドウ制御
├── screenshot.py        # スクリーンショット撮影
├── page_compare.py      # 画像比較（停止判定）
├── pdf_generator.py     # PDF生成
├── config.py            # 設定管理
├── requirements.txt     # 依存パッケージ
├── screenshots/         # 撮影画像保存先
│   └── {book_name}/
└── output/              # PDF出力先
```

## リスク・注意事項

- Kindleアプリのアップデートでウィンドウ構造が変わる可能性
- フルスクリーン切り替えのタイミングでキャプチャが失敗する場合がある
- macOSのバージョンによりアクセシビリティAPIの挙動が異なる場合がある
- ページ描画に時間がかかる本（画像多め）は待機時間の調整が必要
